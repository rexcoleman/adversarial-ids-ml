## 1) Purpose

Documents all configurable parameters for the adversarial IDS project. Config is embedded in source (no external YAML) due to project scope.

## 2) Models

| Model | Key Hyperparameters | Source |
|-------|--------------------|--------|
| Random Forest | n_estimators=100, max_depth=20, min_samples_leaf=5 | EXPERIMENT_CONTRACT §6 |
| XGBoost | n_estimators=100, max_depth=8, learning_rate=0.1, eval_metric=mlogloss | EXPERIMENT_CONTRACT §6 |
| MLP | hidden_layer_sizes=(256,128,64), max_iter=200, early_stopping=True | EXPERIMENT_CONTRACT §6 |

## 3) Seeds

| Parameter | Value |
|-----------|-------|
| Default seed | 42 |
| Stability list | [42, 123, 456, 789, 1024] |
| Seeding call | `set_seed(seed)` before every experiment |

## 4) Epsilon Schedule

Attack perturbation magnitudes (L-infinity norm):

```python
EPSILON_SCHEDULE = [0.01, 0.05, 0.1, 0.2, 0.3, 0.5]
```

Defined as constant in `src/adversarial_attacks.py`.

## 5) Feature Controllability Split

| Category | Count | Description |
|----------|-------|-------------|
| Controllable | 57 | Features an attacker can modify (packet-level) |
| System (defender-observable) | 14 | Features controlled by network infrastructure |
| Total | 71 | Post-cleaning feature set (78 raw → 71 after correlation filter) |

Mask generated by `get_controllable_feature_mask()` in `src/preprocessing.py`.

## 6) Data Sampling

| Parameter | Value |
|-----------|-------|
| Sample fraction | 0.1 (10% of CICIDS2017 = ~283K rows) |
| Split ratios | 70/15/15 (train/val/test) |
| Stratification | By attack type label |

## 7) Defense Parameters

| Defense | Key Parameters |
|---------|---------------|
| Adversarial training | 50K noise-augmented samples, retrain from scratch |
| Feature squeezing | 4-bit depth quantization |
| Constraint-aware detection | Flag if any system feature changed beyond threshold |

## 8) Resolved Config

Every run writes `config_resolved.yaml` to its output directory. See ARTIFACT_MANIFEST_SPEC §2.

## 9) Change Control

Any parameter change requires a `CONTRACT_CHANGE` commit and re-running affected experiments.
